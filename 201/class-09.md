# Forms
- users input information into forms which get submitted via a button to a server 
- The server creates a new page to send back to the browser based on the information received.
- a form can have several form controls with each gather different things
- information is sent from the browser to the server using name/value pairs
- form elements collect information
- HTML5 introduces new form elements which make it easier for visitors to fill in forms
# Lists, Tables, and Forms
- list-style-type and list-style image can be used to change the appearance of list markers
- table cells can have different borders and spacing depending on the browser, bus you can use certain properties to control that 
- Forms are easier to use if the form controls are vertically aligned using CSS
# Events
- HTML elements nest inside other elements, if you hover or click on a link, you will also be hoverin gor clicking on its parent elements
- the flow of eevents only really matters when your code has event handlers on an element and one of its ancestors or descendant elements
- the event object can tell you where the cursor was positioned when an event was triggered
- events are the browsers way of indicating when something has happened 
- binding is the process of stating which event you are waiting to happen, and which event you are waiting for that event to happen upon 
- events can trigger functions
- event delegation can be used to monitor for events
- most common events are W3C DOM
